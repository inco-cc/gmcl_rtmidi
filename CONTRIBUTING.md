# Contributing

Contributions are welcome, but please read the following before [submitting a pull request](https://github.com/inco-cc/gmcl_rtmidi/pulls):

* Please give your pull request a meaningful title that summarizes the changes being made. Additionally, the description should provide further details on said changes, as well as why they should be merged.
* Please refrain from submitting pull requests that contain only superficial changes. Generally speaking, any pull request that introduces source code changes should contain actual functional changes that improve the project in a meaningful way. For documentation, changes that improve clarification, spelling and grammar are preferable.
* Pull requests that introduce source code changes are expected to compile and run without any errors or obvious regressions. Please test your changes locally before submitting a pull request. You will be asked to correct any errors or regressions before your changes will be considered for merging. If your pull request is a work in progress, you can create it as a draft before it is ready for review.
* When introducing new or modified source code changes, please keep it consistent with the existing source code. This project has a [.editorconfig](.editorconfig) file to assist with this. If your changes do not follow the established style of the file being changed, you will be asked to correct this before your changes will be considered for merging.
